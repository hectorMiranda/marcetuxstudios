July had a theme: transitions. The API versioning work is about managing the transition
for existing clients without freezing the API in place. The Kubernetes namespace
reorganization is a structural transition that pays off later when the cluster is larger.
The AngularJS migration is the slowest transition of them all, but the only one that
doesn't require a freeze.

The Bitcoin post got more response than anything I've written in months, mostly from
people who wanted to know if I thought they should buy. I don't have a view on price,
which is not a satisfying answer. The technology is interesting; the speculation is
something else. Those are separable questions and I'll keep them separated.

August is going to be significant: .NET Core 2.0 is releasing this month, which is a
real milestone — the API surface is large enough now that most .NET Framework code can
target Core with minimal changes. I want to run the migration guide against the oldest
internal services and see what the actual delta looks like, not the blog-post version.
Also: rumors of a new role are circling that I'll have more to say about in the fall.
