I'm leaving SolidCommerce at the end of September. A healthtech staffing startup called
Go RN — the Cordova mobile project I've been involved with since spring — offered me
the CTO role, and after a few weeks of thinking I said yes. It's the first time I'll
have a title that doesn't have the word "engineer" in it, which is worth writing about
even if I'm not sure how the writing will go.

The offer to consider was compelling because the technical foundation is already
something I helped shape: the Angular frontend, the Node backend, the Cordova mobile
app. I'm not inheriting code I don't understand; I'm stepping into architect ownership
of code I've been contributing to. The difference between "contractor who built this"
and "CTO who owns this" is accountability for decisions I haven't made yet, not just
the ones I already did. That's the part I've been thinking about.

The practical reality: it's a startup with a small engineering team, which means the
CTO still writes code — every day, probably. This is not the kind of CTO role where
you attend strategy meetings and delegate everything technical. It's a role where you
build the thing, make the hiring calls that shape what you'll be able to build in six
months, and think about architecture with a budget constraint attached to every decision.
I've been preparing for that without knowing it. September is the last month at
SolidCommerce; I'll write more from the new vantage point in October.
