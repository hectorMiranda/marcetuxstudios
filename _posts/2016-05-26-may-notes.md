May went fast. The TypeScript 2.0 migration is done and the discriminated union types
are paying off in code review — fewer "did you remember to check for the error case"
comments and more "the compiler caught this." HLS packaging is in production and the
stall rate on short-form content is down. Not dramatic — our clips are short enough
that progressive download was mostly fine — but measurably better on slower connections.

IAM least privilege cleanup was the least exciting work of the month and probably the
most valuable — not because of an incident, but because the old policy was an accident
waiting to happen and now it isn't. React at the margins is a pattern I'll keep using:
islands of component-based UI in an existing codebase, no rewrite required.

June is going to be interesting from a .NET perspective — .NET Core 1.0 should ship
this month and I want to get a real project running on it, not just the "hello world"
I've run from the RC. The question is whether the tooling story is actually production-
ready or if it's still a preview dressed up as a release.
