Last post of the year, shorter than usual because it's the holidays and I've been spending time away from the keyboard. The .NET Core 3.1 LTS adoption plan is drafted for the Q1 planning review. The LoRa sensor node for the outdoor circuits is sketched on paper but not started in Altium yet — that's January's project.

A few things I've been reading and thinking about that will probably show up as posts in early 2020: the W3C trace context spec becoming more widely adopted (the `traceparent` header is now an official standard), what GitOps looks like for Kubernetes deployments and whether it changes how we think about pipelines, and the Kubernetes operator pattern for stateful services. None of those are urgent; all of them are the kind of thing you want to understand before you need to.

The decade ends quietly here. The 2010s were the decade where the cloud became real, containers went from curiosity to infrastructure, and .NET stopped being Windows-only. The 2020s start with all of that settled and a new set of questions about what you build on top of it. I'm looking forward to the questions.
